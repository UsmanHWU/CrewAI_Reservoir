# 🛢️ Reservoir Well Production Analyzer
### A CrewAI Multi-Agent Project for Petroleum Engineering

> **Course:** Agentic AI Design | **Domain:** Petroleum / Reservoir Engineering

---

## What This Project Does

This project uses **three AI agents** to automatically analyze oil well production data and generate an engineering report with actionable recommendations.

Given a simple text file of well data, the crew will:
1. **Read and extract** all key reservoir parameters
2. **Calculate** Productivity Index (PI), Recovery Factor (RF), and Decline Rate
3. **Write** a professional well performance report with ranked recommendations

## Setup Instructions

### Step 1: Clone and enter the project
```bash
git clone <your-repo-url>
cd reservoir_crew
```

### Step 2: Create a virtual environment
```bash
python -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate
```

### Step 3: Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4: Set up API keys
```bash
cp .env.example .env
# Open .env in a text editor and fill in your keys
```

You need:
- **OpenAI API key** → required (get at https://platform.openai.com)
- **Serper API key** → optional (for web search, get at https://serper.dev)
- **Langfuse keys** → optional (for monitoring, get at https://cloud.langfuse.com)

### Step 5: Run the project
```bash
python main.py
```

The report will be saved to `outputs/well_report.md`.

## The Three Agents

| Agent | Role | Tools Used |
|---|---|---|
| **Data Scout** | Reads well data file, extracts all parameters | FileReadTool, SerperDevTool |
| **Reservoir Analyst** | Calculates PI, RF, Decline Rate | ReservoirCalculatorTool (custom) |
| **Report Writer** | Writes the final engineering report | None (uses context from other agents) |

---

## The Three Tools

### Tool 1: FileReadTool (Pre-built)
Reads `data/well_data.txt` and returns its contents for the Data Scout.

### Tool 2: SerperDevTool (Pre-built)
Searches the web for production benchmarks or field-specific context (e.g., typical PI for Seria Sand).

### Tool 3: ReservoirCalculatorTool (Custom ⭐)
Located in `tools/reservoir_calculator.py`. Takes well parameters as input and calculates:

| Metric | Formula | Unit |
|---|---|---|
| Productivity Index (PI) | q / ΔP | STB/day/psi |
| Recovery Factor (RF) | Np / OOIP × 100 | % |
| Decline Rate (D) | -ln(qf/qi) / t | /year |

---

## Fallback Mechanism

The project handles failures at two levels:

1. **Tool-level fallback** (in `reservoir_calculator.py`): If parameter parsing fails, the tool returns industry rule-of-thumb estimates instead of crashing.

2. **LLM-level fallback** (in `main.py`): If the primary LLM (GPT-4o-mini) fails, `run_with_fallback()` automatically switches all agents to GPT-3.5-turbo and retries.

**To test the fallback:** Set `OPENAI_API_KEY=fake-key` in `.env` and run the project. You'll see the fallback activate in the console.

## MCP (Model Context Protocol) — How It Could Help

MCP is a standard protocol for connecting agents to external tools and data sources. Here's how this project would benefit:

| Current Approach | With MCP |
|---|---|
| Read one local text file | Connect to a live SCADA/production database |
| Hardcoded calculator tool | Hot-swap to a company's internal engineering API |
| Manual well data entry | Agent reads real-time sensor data via MCP server |
| Web search for benchmarks | Connect to SPE OnePetro or IHS Markit MCP server |

For example, a **MCP SQL Server** could let the Reservoir Analyst query 10 years of production history from a database instead of reading a static text file — without any code changes to the agent.

## Key Petroleum Engineering Concepts Used

- **Productivity Index (PI):** Measures how easily fluid flows from reservoir to wellbore. Higher = better well.
- **Recovery Factor (RF):** What percentage of the OOIP has been produced. Typical: 15–30% for solution gas drive.
- **Exponential Decline Rate (D):** How fast the well's production is declining per year.
